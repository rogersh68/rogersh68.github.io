Chapter 6: The Document Object Model

DOM intro

  • represents an HTML document as a network of connected notes that form a tree-like structure.
  • provides several methods that allow us to access any element on a page.

Getting elements

  • use document.name to access elements of a page.
  • Shortcut Methods
    • document.body - returns the body element of a web page
    • document.images - returns a node list of all the images
    • etc.
  • document.getElementById(), getElementsByTagName(), getElementsByClassName(), querySelector(), querySelectorAll()

Navigating the DOM

  • can use properties like childNodes, paredNode, firstChild, nextSibling, etc.

Getting and setting an element's attributes

  • The "getAttribute()" method returns the value of the attribute provided as an argument
  • The "setAttribute()" takes two arguments: the attribute you wish to change, and the new value.

Updating the DOM

  • const variable = document.createElement('li'); variable.textContent = 'variable';

Changing the CSS of an element

  • variable.style.property = "vlaue";

Chapter 7: Events

Events intro

  • An event listener will let the program know when an event happens, then the program can respond.
  • Allows programs to continue with the program without constantly checking for the event.

Adding event listeners

  • Example: document.body.addEventListener('click', doSomething);
  • when a user 'clicks' on the body the program will run doSomething()

The Event Object

  • event.type - returns the type of event that occurred
  • event.target - returns a reference to the node that fired the event
  • event.pageX, event.pageY - show the number of pixels from the left and top where the event took place in the document

Mouse, keyboard and touch events

  • Mouse: click, mousedown, mouseup, dblclick, mouseover, mouseout, mousemove
  • Keyboard: keydown, keypress, keyup
  • Touch: touchstart, click (300ms delay), touchend, touchmove, touchenter, touchleave, touchcancel

Removing event listeners

  • removeEventListener()

Stopping default behavior

  • preventDefault()

Event propagation

  • The order event fire on each element
  • Bubbling: the even fires on the element clicked on first, the bubbles up the document tree, firing an event on each parent element until it reaches the root node.
  • Capturing: starts by firing an event on the root element, then propagates downwards, firing an even on each child element until it reaches the target element that was clicked on.
