Chapter 10: Testing and Debugging

Errors, exceptions, and warnings

  • Errors are caused when soemthing goes wrong in a program via a system error, programmer error, or user error.
  • Exceptions are errors that produce a return value that can be used by the program to deal with the error.
  • Warnings can occur if there's an error that isn't enough to cause the program to crash. The program will continue running.

Strict mode

  • Produces more exceptions and warnings. Makes it easier to spot errors early on.
  • Use by adding 'use strict' to the first line of a JS file. (create function)

Debugging in the browser

  • Use different ways to create breakpoints in your code so you can stop and see values etc.
    • alert()
    • console.log()
    • console.trace()
  • Debugging tools are provided in most modern browsers.
    • Use 'debugger' keyword to make a breakpoint. Remember to remove.

Error objects

  • const error = new Error('Error message');
  • Error objects for specific errors:
    • EvalError
    • RangeError - a number is outside an allowable range of values.
    • ReferenceError - a reference is made to an item that doesn't exist.
    • SyntaxError - there's an error in the code's syntax.
    • TypeError - there wrong type of value is used.
    • URIError - there's a problem encoding or decoding the URI.
    • InternalError - error occurs in the JS engine
  • Error object properties that are generally safe to use:
    • name - returns the name of the error constructor function.
    • message - returns a description of the error.
    • stack - returns a stack trace of the error.

Throwing exceptions

  • Create your own exception with a 'throw' statement.
  • You can throw an error object.

Exception handling

  • If we suspect a piece of code will result in an exception, we can wrap it in a try block. This will run the code inside the block as normal, but if an exception occurs it will pass the error object that is thrown onto a catch block.
  • The code inside the catch block will only run if an exception is thrown inside the try block.
  • A finally block can be added after a catch block. This will always be executed after the try or catch block, regardless of whether an exception occurred or not.

Testing frameworks

  • You can write your own tests or use a testing framework.
  • Ex: Jest
